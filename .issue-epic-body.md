## Objetivo

Implementar e validar o catálogo aberto de agentes do AgentsKit Registry — **346 specs** no manifest, **36 já validados**, **310 drafts** pendentes.

Cada sub-issue abaixo cobre **uma vertical** com a spec completa de cada agente: o que é, para que serve, gates e checklist de implementação.

## Contrato mínimo (todos os agentes)

- Uma dor → um output tipado (Zod)
- Provider-agnostic (`adapter` no config)
- `agent.ts` + `meta.json` + `agent.test.ts` + `README.md` + `eval.ts` (antes de validar)
- `status: draft` no catálogo → `status: validated` só após testes + curadoria
- Bloqueados: ver `catalog/content-policy.json`

## Stacks (workflows)

- **`stack-coding-ship`**: PRD → issues → specs → implement → QA → review → release notes. (7 agentes)
- **`stack-marketing-campaign`**: Brief → competitive intel → copy → review → schedule → publish → digest. (7 agentes)
- **`stack-agency-client-delivery`**: Kickoff brief → creative → deck → schedule → status report. (6 agentes)
- **`stack-support-ticket`**: Triage → KB match → macro or escalation draft. (4 agentes)
- **`stack-fintech-onboarding`**: KYC → sanctions → transaction monitor → investigator. (3 agentes)
- **`stack-clinical-intake`**: Triage → summary → SOAP → referral routing. (4 agentes)
- **`stack-legal-matter`**: Case analysis → doc review → redaction → drafting. (4 agentes)
- **`stack-sales-outbound`**: Lead score → research → personalized sequence → call prep → debrief. (5 agentes)
- **`stack-devops-incident`**: Alert triage → runbook match → IC aide → postmortem → action tracker. (5 agentes)
- **`stack-ecosystem-doc-bridge`**: Dogfood: scan corpus → classify memory candidates → handoff → promote to public docs. (4 agentes)
- **`stack-ecosystem-registry-growth`**: Dogfood: draft new agent spec → scaffold → eval author → playbook alignment check. (3 agentes)
- **`stack-compliance-lgpd`**: LGPD assessment → DPA review → retention plan → breach notification draft (BR). (4 agentes)
- **`stack-hr-hire`**: JD → screen → interview bank → debrief → offer. (5 agentes)
- **`stack-data-analyst`**: Question → SQL → review → narrative report. (4 agentes)
